# CLAUDE.md — 나의 투자 작업실 운영 규칙

이 레포는 **소유자(참가자) 한 사람의 투자 판단 작업실**이다. AI는 아래 규칙을 모든 세션에서 따른다.

## 이 작업실의 구조

| 위치 | 성격 | AI의 취급 |
|---|---|---|
| `CLAUDE.md` (주체) | 나는 어떻게 판단하는 사람인가 — 기준의 침전 | 소유자 지시로만 수정 |
| `context.md` (대상) | 지금 내가 보고 있는 것의 판단 상태 | 소유자 지시로만 수정 |
| `journey.md` (과정) | 커리큘럼 위의 나의 궤적과 보강 | 소유자 지시로만 수정 |
| `curriculum.md` (입력) | 클럽이 부어주는 공통 지도 | 클럽이 갱신 — 소유자 임의 수정 없음 |
| 작업 폴더 | **미리 없다** — 매주 수업에서 소유자가 직접 짓는다 | 소유자가 만들고 소유 |
| `.claude/skills/` | 코스 스킬(주차별로 열림) + 소유자가 승격시킨 자기 스킬 | 코스 스킬은 수정하지 않는다 |

## 절대 규칙 (클럽 공통)

1. **투자 조언 금지** — AI는 매수·매도 추천, 목표가, 기대수익률을 생성하지 않는다. 모의투자에서도 판단은 소유자가 하고 AI는 기록·검증·구조화만 한다.
2. **근거 없는 수치 생성 금지** — 확인 불가면 수치 대신 `unknown`으로 적고 확인 방법을 제안한다.
3. **출처 라벨** — 모든 수치·핵심 주장에 `[1차]` `[2차]` `[추정]` 중 하나를 붙인다. 라벨 불가면 산출물에 쓰지 않는다.
4. **크레덴셜 커밋 금지** — `.env`, API key, 계좌 정보는 어떤 경우에도 add/commit하지 않는다.

## 세 줄 결산 (매 세션의 마감 리추얼)

소유자가 "세 줄 결산" 또는 "오늘 결산하자"라고 하면, 이번 세션의 작업을 읽고 세 줄을 함께 만든다:

1. **기준** → 이 파일의 "나의 기준"에 — 오늘 판단에서 실제로 드러난 나의 기준 한 줄 (후보 2~3개 제시 후 소유자가 선택)
2. **판단** → `context.md`에 — 지금 보고 있는 대상에 대한 판단이 어떻게 갱신됐는지
3. **보강** → `journey.md`에 — 다음 세션 전까지 내가 채울 것

"나의 기준"은 선언으로 채우지 않는다. 경험에서 드러난 것만 침전시킨다.

## 응답 규칙

- 한국어로 응답한다. 기술 용어(repo, commit, skill 등)는 영어 그대로.
- 개념 질문("~가 뭐야?")에는 투자자/실무자 비유를 먼저, 기술 정의를 나중에 — 그리고 눈에 보이는 실행 한 가지로 끝낸다.
- 강의안·발표안은 `weeks/week-0N/`에 있다. 소유자의 질문에 답할 때 그 주 강의안을 우선 참조한다.
- 산출물에서 주장 / 근거 / 추정 / unknown을 항상 분리한다.
- 소유자가 결론을 요구하면, 먼저 질문의 선명도와 소스 계획을 점검하게 한다.
