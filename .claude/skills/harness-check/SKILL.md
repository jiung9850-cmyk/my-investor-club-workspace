# Harness Check

참가자의 작업면이 "AI가 이어서 일할 수 있는 상태"인지 6문항으로 진단한다. 제출 전과 주중 점검에 사용한다.

## 진단 절차

레포 루트의 `context.md`, `CLAUDE.md`와 `outputs/`를 읽고 아래 6문항을 평가한다. 파일을 읽지 않고 참가자 자기보고만으로 O를 주지 않는다.

| # | 축 | 질문 | O 기준 | X 신호 |
|---|---|---|---|---|
| 1 | 질문 선명도 | 메커니즘 질문 v2가 검증 가능한가 | 6축 태그 2+ 와 반증 조건이 있다 | "유망한가?" 류 방향성 질문 |
| 2 | 맥락 구조 | context.md 7줄이 현재 상태를 반영하는가 | 7줄 전부 채워짐 + 최근 세션 반영 | 빈칸, 또는 2주 전 상태 그대로 |
| 3 | 소스 계획 | 다음 소스 액션이 실행 가능한가 | 30분 내 착수 가능한 행동 3개 | "더 리서치한다" |
| 4 | 실행 | AI가 지금 바로 할 작업이 정의되는가 | 이 파일들만 읽고 첫 작업을 특정할 수 있다 | 참가자 구두 설명 없이는 시작 불가 |
| 5 | 검증 | 산출물에 주장/근거/추정 분리가 있는가 | 수치 전부 라벨, verifier 통과 이력 | 라벨 없는 수치 존재 |
| 6 | 개선 | 규칙이 경험으로 갱신되고 있는가 | 금지 가정·결정 규칙에 갱신 흔적(커밋) | Week 1 이후 규칙 변화 없음 |

△는 "구조는 있으나 품질 미달"일 때 준다 (예: 금지 가정이 3개 있지만 전부 일반론).

## 출력 형식

```text
Harness Check — {ID} / {날짜}

1. 질문 선명도   O   {한 줄 근거}
2. 맥락 구조     △   {무엇이 미달인지}
3. 소스 계획     O
4. 실행          X   {AI가 막히는 지점 인용}
5. 검증          △
6. 개선          O

Quick Fix 3 (효과 큰 순):
  1. {15분 내 가능한 수리}
  2. ...
  3. ...

다음 주차 연결: 이 상태로 Week {n} 과제를 시작할 수 있는가 — {가능/보완 후 가능}
```

## 규칙

- 4번(실행)의 평가는 실제로 시뮬레이션한다: "이 파일들만 읽은 새 담당자"로서 첫 작업을 세워 보고, 막히면 막힌 지점을 인용한다
- 6문항 전부 O여도 칭찬으로 끝내지 않는다. 가장 약한 축 하나에 대해 다음 단계 제안을 붙인다
- X가 3개 이상이면 quick fix 대신 해당 주차 skill(`/context-layer-builder` 등) 재실행을 권한다
