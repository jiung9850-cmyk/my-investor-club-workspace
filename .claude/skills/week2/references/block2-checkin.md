# Block 2: 첫 스킬 = check-in (floor)

## EXPLAIN

이제 **인생 첫 스킬**을 만듭니다. 겁먹지 마세요 — 스킬은 코드로 짜는 게 아니라 **한 문장으로 시키는** 것입니다. W1에서 한 위임 그대로예요.

왜 하필 만드나? 남이 만든 MCP·도구만으론 "매주 내 방향을 점검하는 나만의 루틴"이 안 됩니다. 도구를 **쓰는** 데서 **만드는** 데로 넘어가는 순간이 오늘입니다.

**체크인 스킬**은 매주 여러분의 하네스를 점검하는 스킬입니다 — 내 선언과 이번 주 지은 것을 읽고, "내 방향이 어디로 움직였나"를 되물어 줍니다.

## EXECUTE

1. 아래를 그대로 입력해 스킬을 짓게 하세요 (Claude가 SKILL.md를 만들어 줍니다):

```
매주 내 방향을 점검하는 check-in 스킬을 만들어줘.
- journey.md 의 "나의 선언" 과 이번 주 outputs/ 를 읽고,
- 내 방향이 지난주와 비교해 어떻게 움직였는지 물어봐 주고,
- 내가 답하면 journey.md 에 이번 주 점검 한 줄을 남겨줘.
파일은 .claude/skills/check-in/ 에 저장해줘.
```
(중간에 `Allow?`가 뜨면 Enter)

2. 만들어지면 **한 가지만 내 식으로 추가**하세요 — 예: "이번 주 강연에서 내 방향에 걸린 게 있었나?"도 물어봐 줘.

3. **바로 돌려봅니다** (dogfood): 
```
/check-in
```

## CHECK

- `.claude/skills/check-in/SKILL.md` 가 생겼는지 확인한다.
- `/check-in`을 실행했을 때 무엇이 나오고 무엇이 바뀌었는지 눈으로 본다 — **"스킬을 실행한다는 게 이거였구나."**
- 좋은 체크인 기준: *다음 주의 내가 이것만 돌리면 방향 점검이 되는가?*
- 선언한다: **"여기까지가 오늘의 완주입니다.** check-in 하나면 성공이에요. 여기서 멈춰도 됩니다."
