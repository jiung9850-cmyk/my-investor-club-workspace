# Mechanism Question Builder

관심사("이 회사 괜찮은 것 같다")를 판단 가능한 질문으로 변환하는 skill. **절대 투자 판단에 답하지 않는다.** 이 skill의 산출물은 답이 아니라 질문이다.

## 1단계 — 인터뷰 (한 번에 하나씩)

다음을 순서대로 묻는다. 이미 대화나 `context.md`에 답이 있으면 건너뛴다.

1. 관심 대상은 무엇인가 (기업/테마/섹터 — 한 줄로 특정)
2. 투자 시계는 어느 정도인가
3. 이 대상에 대해 **이미 믿고 있는 것**은 무엇인가 (이것이 편향의 재료다)
4. 접근 가능한 소스는 무엇인가 (공시를 볼 수 있는 상장사인가, IR 자료뿐인 비상장사인가)

## 2단계 — 6축 매핑

질문은 아래 6축 중 **최소 2개**를 동시에 건드려야 한다. 참가자에게 표를 보여준다.

| 축 | 질문의 재료 |
|---|---|
| 수익 | 누가, 왜, 얼마를 내는가. 매출의 단위경제 |
| 비용 | 원가 구조는 어디서 오는가. 규모에 따라 어떻게 변하는가 |
| 자본 | 성장에 얼마의 돈이 필요한가. 누가 대는가 |
| 위험 | 어떤 경로로 죽는가. 규제/집중도/의존성 |
| 시장기대 | 가격에 이미 무엇이 반영되어 있는가 |
| 회수 | 어떻게 돈이 돌아오는가. 인수 수요/상장/배당 |

## 3단계 — 질문 후보 5개 생성

각 후보는 다음 형식을 지킨다.

```text
Q{n}. {질문 한 문장} [축1][축2]
  - 필요한 소스 유형: {예: 공시 원가 내역 [1차], 경쟁 제품 가격 [2차]}
  - 반증 조건: {무엇이 확인되면 이 질문의 전제가 무너지는가}
```

**실격 기준** — 생성 후 스스로 검사하고, 걸리면 그 후보를 버리고 다시 만든다.

- "성장할까 / 유망할까 / 좋은 회사일까" 류의 방향성 질문 (어떤 답이 나와도 행동이 안 바뀜)
- 축 태그가 1개 이하인 질문
- 반증 조건을 쓸 수 없는 질문 (반증 불가능하면 신앙이지 가설이 아니다)

## 4단계 — 질문형 목차 3계층 배치

고른 질문이 판단 문서의 어디에 놓이는지 보여준다. 좋은 심사 문서의 목차는 명사("시장 분석")가 아니라 질문이다.

```text
① 시장 — 신규 진입자/투자자에게 충분히 매력적인 시장인가?
② 제품 — 경쟁력 있는 제품/서비스를 만들 수 있는가? 그 우위는 방어되는가?
③ 판매 — 좋은 제품을 어떻게 팔 것인가? 반복 가능한 판매 구조인가?
```

참가자의 질문 v2가 ①~③ 중 어디에 속하는지 표기하고, 나머지 두 계층에서 **아직 비어 있는 질문**을 unknowns 후보로 제시한다.

## 5단계 — v2 확정

참가자가 후보 1개를 고르면 두 가지를 되물어 좁힌다.

1. "이 답이 나오면 당신의 행동이 실제로 무엇이 달라집니까?"
2. "한 번의 리서치 사이클(1주)로 반증 시도가 가능한 크기입니까?" — 크면 자르고, 잘린 조각은 unknowns로 보낸다

최종 출력: 질문 v2 + 축 태그 + 반증 조건 + 첫 소스 액션 1개. `context.md`가 있으면 2번 줄에 반영을 제안한다.

## 좋은 예 / 나쁜 예

- 나쁨: "산업용 로봇 부품 시장이 유망한가?"
- 좋음: "이 부품사의 원가 우위 `[추정]` 는 설계 구조에서 오는가, 저임금 외주에서 오는가? [비용][위험] — 반증: 원가 절감분의 과반이 인건비 항목이면 복제 가능한 우위다"
