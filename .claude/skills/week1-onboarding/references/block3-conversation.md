# Block 3: 대화 — 읽게 하기, 질문시키기

## EXPLAIN

**수업 복습**: "AI가 스스로 객관식으로 물어보게 할 수 있습니다 — ask user question. 뭘 지시할지 모르겠으면 AI에게 물어보게 하세요."

모호한 요청과 명확한 대화의 차이를 몸으로 확인한다:

| 모호하게 시키면 | 질문하게 하면 |
|---|---|
| AI가 혼자 추측해서 그럴듯한 결과 | AI가 나에게 물어보고, 내 답으로 만든 결과 |

## EXECUTE

세 가지를 순서대로:

1. **읽게 하기**: `curriculum.md 읽고, 5주 중 내가 가장 기대할 만한 것 하나를 골라 이유와 함께 말해줘`
2. **모호하게 시켜보기** (일부러): `이번 클럽 잘 다니게 도와줘` — 결과가 그럴듯하지만 내 것이 아님을 관찰한다
3. **질문시키기**: `내가 이 클럽에서 뭘 얻고 싶은지 파악하려면 나에게 뭘 물어봐야 해? AskUserQuestion으로 하나씩 물어봐줘` — 답들은 Block 6(선언)의 재료가 되니 성실히 답한다

## QUIZ / 스캐폴드 확인

[스캐폴드] 3번의 문답 요약을 journey.md 1주차 행("한 일 / 남긴 것")에 기록하게 한다.

퀴즈: "2번과 3번의 차이는 무엇이었나요?"
- **3번은 내 맥락이 들어가서 결과가 내 것이 됐다** (정답 — 맥락 없는 AI는 평균적인 답밖에 못 한다)
- 3번이 더 빨랐다 (속도는 비슷하거나 더 느림)
- 2번이 틀린 답을 했다 (틀린 게 아니라 '누구의 것도 아닌' 답)
