# Block 1: Why — 왜 터미널인가

## EXPLAIN

**수업 복습**: "왜 개발자가 아닌 우리가 터미널에서 일해야 하는가 — 이것을 기술 해방이라고 부릅니다. GUI는 버튼만 누를 수 있지만, CLI는 컴퓨터의 모든 것을 할 수 있습니다."

```
GUI:  사용자 → [버튼] [버튼] [버튼] → 제한된 기능
CLI:  사용자 → 자유로운 명령어 ────→ 컴퓨터의 모든 기능
      AI 에이전트 → CLI ──────────→ 소프트웨어 직접 제어
```

이 블록은 설명 대신 퀴즈 2개로 진행한다 (Phase A에서 Quiz 1, Phase B에서 Quiz 2).

## QUIZ-1

"왜 굳이 터미널을 깔아서 Claude Code를 사용해야 할까요?"
- **CLI는 컴퓨터의 모든 일을 할 수 있어서** (정답 — GUI는 버튼만, CLI는 제한 없음)
- 터미널이 더 예뻐서 (예쁜 건 GUI가 더 예쁨)
- 프로그래머처럼 보이려고 (멋은 부산물)
- 시켜서 (아무도 안 시킴)

정답 해설에 위 도식을 다시 보여준다.

## QUIZ-2

"수업에서 '전원이 이미 AI 사용자'라는 설문 결과를 봤습니다. 그런데 왜 채팅 서비스가 아니라 이걸 배울까요?"
- **채팅은 답을 주고 끝나지만, 여기서는 판단의 기록이 파일로 남아 다음 세션이 이어받아서** (정답 — 수업의 핵심: 이어받을 수 있는 작업환경)
- 채팅보다 답이 더 정확해서 (모델은 같다)
- 무료라서 (둘 다 유료)
- 투자 정보가 더 많아서 (정보량의 문제가 아님)

정답 피드백: "그 '이어받는 파일들'이 여러분 작업실의 네 파일입니다 — 다음 블록부터 직접 짓습니다."
