# Block 4: 모드 — plan과 auto

## EXPLAIN

**수업 복습**: "shift+tab을 누르면 아래 모드가 바뀝니다 — 전부 허락받는 모드, 자동 수락(accept edits), 플랜 모드(먼저 계획을 논의), 오토(자율주행). 저는 보통 플랜 아니면 오토로 씁니다."

| 모드 | 언제 |
|---|---|
| plan mode | 일이 크거나 애매할 때 — 계획을 먼저 보고 승인 |
| accept edits / auto | 익숙한 반복 작업 — 맡기고 결과만 확인 |

이 감각(위임의 강도를 내가 조절한다)이 곧 "권한 설정 = 위임 계약"이다.

## EXECUTE

1. **shift+tab**을 눌러 화면 아래 모드 표시가 바뀌는 것을 확인한다 (몇 번 눌러 한 바퀴 돌아본다)
2. **plan mode** 상태에서 시켜본다: `journey.md 진행 상태 표의 1주차 행을 오늘 한 것들로 채우려고 해. 계획부터 보여줘`
3. 계획이 먼저 오는 것을 확인 → 승인 → 실행되는 흐름을 경험한다

## QUIZ / 스캐폴드 확인

[스캐폴드] journey.md 1주차 행이 채워졌는지 확인시킨다.

퀴즈: "plan mode는 언제 쓰는 게 좋을까요?"
- **일이 크거나 뭘 시켜야 할지 애매할 때** (정답 — 계획을 먼저 합의하면 결과가 어긋나지 않는다)
- 항상 (반복 작업엔 비효율)
- 절대 안 씀 (자율주행만 쓰면 큰 작업에서 방향이 어긋남)
