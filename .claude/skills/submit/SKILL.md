# Submit — 과제 제출

제출을 한 번에 처리하는 skill. 각 단계가 실패하면 **이후 단계를 실행하지 않는다** (fail-fast). 참가자는 git을 몰라도 된다 — 이 skill이 순서를 보장한다.

## Step 0 — 워크스페이스 정리

1. `git status`로 변경 파일을 확인한다
2. **경계 검사**: 변경이 `club/` 아래에 있으면 **중단**하고 보고한다 — `club/`은 교재라 수정하지 않는다. 되돌릴지 물어본 뒤 처리한다 (`git restore` 안내)
3. 변경사항이 아예 없으면 "제출할 변경이 없습니다"라고 보고하고 종료한다
4. 제출 대상 파일 목록을 보여주고 참가자 확인을 받는다

## Step 1 — 크레덴셜 차단

커밋 전에 반드시 검사한다. 하나라도 걸리면 해당 파일을 **제외**하고 경고한다.

- 파일명 패턴: `.env*`, `credentials*`, `*.key`, `*.pem`, `*token*`
- 내용 패턴: 변경 파일 내 `api_key`, `secret`, `password`, `Bearer `, 계좌번호로 보이는 숫자열
- 개인정보: 타인 실명+연락처 조합이 보이면 경고 후 참가자 판단을 받는다

## Step 2 — Commit

1. 세션에서 수행한 작업 기반으로 한국어 커밋 메시지를 생성한다. 형식: `week{n}: {ID} {한 줄 요약}` (예: `week1: hong-gildong context.md 7줄 및 금지 가정 제출`)
2. Step 1을 통과한 파일만 `git add` → `git commit`

## Step 3 — Pull & Push

27명이 한 레포를 쓰므로 push 전 동기화가 필수다.

1. `git pull --rebase origin main`
2. **충돌 발생 시**: 중단하고 충돌 파일을 보고한다. 충돌이 본인 폴더 내 파일이면 해결을 도와주고, 타인 폴더면 `git rebase --abort` 후 강사 호출을 안내한다
3. `git push origin main`
4. push 실패 시 에러를 그대로 보여주고 `/github-helper` 절차로 전환한다

## Step 4 — Slack 제출 문안 생성

push 성공 후, `templates/slack-submission-week1.md` 형식(해당 주차 템플릿)으로 제출 문안을 생성해 **복사 가능한 코드 블록**으로 출력한다. 자동 발송하지 않는다 — 참가자가 직접 #과제제출 채널에 붙여넣는다.

문안에 반드시 포함: 이름/ID, 주차, 산출물 경로, 질문 v2 한 줄, 이번 주 가장 큰 unknown 한 줄, commit hash.

## 에러 처리 요약

| 상황 | 동작 |
|---|---|
| 변경 없음 | 보고 후 종료 |
| 본인 폴더 밖 변경 | 중단, 되돌리기 안내 |
| 크레덴셜 감지 | 해당 파일 제외 + 경고 |
| rebase 충돌 (본인 파일) | 해결 지원 후 재개 |
| rebase 충돌 (타인 파일) | abort + 강사 호출 |
| push 실패 | 에러 표시, github-helper 전환 |
| 문안 생성 실패 | git 작업은 완료 유지, 문안만 수동 작성 안내 |

## Week 1 예외 — push가 안 되어도 제출은 성립한다

commit 이후 push가 실패하면(원격 권한/인증 없음 등) **중단하지 말고**:
1. "push는 다음 주에 함께 해결합니다"라고 안내하고
2. Slack 제출 문안은 로컬 커밋 해시 기준으로 그대로 생성한다 (`commit {hash} · push 대기`)
3. 실패 원인을 한 줄로 기록해 두라고 권한다 (다음 주 첫 작업)

Slack 제출이 공식 제출이다. push는 백업이지 관문이 아니다.
